# Glossary

> **Skip if:** you already know what every term in the handbook means.
>
> One-line definitions with a pointer to the chapter that covers each in depth. Alphabetised by the term as it appears in the handbook (not by underlying concept).

## A

### `AnimationFrame`
The driver that pumps high-frequency engine state (playhead position, meters) from the AudioWorklet into the main-thread UI. Without `AnimationFrame.start(window)`, the playhead doesn't move. See [Ch. 03](./03-animation-frame.md).

### `AudioContext`
The browser's root audio object. OpenDAW creates one (typically at 48 kHz) inside `initializeOpenDAW()`. Must be `resume()`d after a user gesture before any sound plays. See [Quick Start](./quick-start.md), [Ch. 12](./12-browser-compatibility.md).

### `AudioFileBox`
The box that holds metadata for one audio file (filename, duration, start/end). Marked as a *preserved* resource — its UUID is content-addressed and stable across projects. See [Ch. 04](./04-box-system-and-reactivity.md), [Ch. 05](./05-samples-peaks-and-looping.md).

### `AudioRegionBox`
A clip placed on a track. Holds position, duration, loop offset, fade in/out, and a `file` pointer to its `AudioFileBox`. See [Ch. 04](./04-box-system-and-reactivity.md), [Ch. 09](./09-editing-fades-and-automation.md).

### `AudioUnitBox`
The "channel strip" container — owns one or more `TrackBox`es plus volume, pan, mute, solo, and an effect chain. One per track-group in the UI. See [Ch. 04](./04-box-system-and-reactivity.md), [Ch. 11](./11-effects.md).

### AudioWorklet
The real-time audio thread where the engine processor actually runs. Constrained to ~2.9 ms per 128-sample render quantum — no `await`, no allocation. See [internals/01](./internals/01-engine-processor.md).

### Adapter
A typed wrapper around a box that exposes its fields with `ValueMapping` / `StringMapping`, plus derived state. `AudioRegionBoxAdapter`, `AudioFileBoxAdapter`, etc. See [internals/02](./internals/02-box-system.md).

### Aux bus
A bus (`AudioBusBox` + `AudioBusProcessor`) that receives audio from one or more aux sends and mixes it. Typically used for reverb/delay returns. See [Ch. 11](./11-effects.md), [internals/05](./internals/05-devices-and-effects.md).

### Aux send
A processor that taps a copy of an `AudioUnit`'s signal (post-effects, pre-channel-strip) and routes it to an aux bus. See [internals/05](./internals/05-devices-and-effects.md).

### Automation
Time-varying parameter values stored in `ValueEvent`s on a `ValueEventCollectionBox`. The audio worklet evaluates the curve once per render quantum (and at event boundaries for sample-accurate changes). See [Ch. 09](./09-editing-fades-and-automation.md), [internals/01](./internals/01-engine-processor.md).

## B

### Box
A persistent graph node — the SDK's data unit. Every track, region, file, effect, automation event is a box. Boxes have fields, pointer rules, and a UUID. See [Ch. 04](./04-box-system-and-reactivity.md), [internals/02](./internals/02-box-system.md).

### Box graph
The set of all live boxes and their pointer relationships. Owned by `Project`. The single source of truth for project state. See [Ch. 04](./04-box-system-and-reactivity.md).

### `BoxIO`
The catalogue dispatch — generated by forge, maps a class name (string) to the matching box class for deserialisation. See [internals/02](./internals/02-box-system.md).

### BPM
Beats per minute. The playback speed. Live BPM lives in `project.timelineBox.bpm` (a `Float32Field`) and is observable. PPQN-to-seconds conversions depend on BPM (and tempo automation if any). See [Ch. 02](./02-timing-and-tempo.md).

## C

### `catchupAndSubscribe`
The `Observable` subscription pattern that fires the callback once immediately with the current value, then again on every change. Use this when late subscribers must not miss the initial state. See [Ch. 03](./03-animation-frame.md).

### Channel strip
The volume / pan / mute / solo stage at the end of an `AudioUnit`'s effect chain. Implemented by `ChannelStripProcessor`. See [internals/05](./internals/05-devices-and-effects.md).

### Clip
The user-facing name for a region the user can click and drag in a UI. In the box graph, a clip is an `AudioRegionBox` or `NoteRegionBox` — *region* and *clip* are nearly synonyms; *clip* is the UI word, *region* is the box word. See [Ch. 04](./04-box-system-and-reactivity.md).

### Comp lane
A take-management lane that holds alternates of the same recorded segment so the user can comp the best parts together. See [Ch. 09](./09-editing-fades-and-automation.md).

### COOP / COEP
`Cross-Origin-Opener-Policy: same-origin` and `Cross-Origin-Embedder-Policy: require-corp`. Together they enable cross-origin isolation, which is required for `SharedArrayBuffer` — which the entire engine state-sync mechanism depends on. See [Ch. 12](./12-browser-compatibility.md).

### `Curve.normalizedAt`
The shared interpolation formula used by automation curves and fades. Single `slope` parameter in [0, 1] (0.5 is linear, < 0.5 logarithmic, > 0.5 exponential). See [Ch. 09](./09-editing-fades-and-automation.md).

## D

### `.dawproject`
The Bitwig-standard interchange format (ZIP of XML plus audio resources). OpenDAW can import and export it via `DawProjectService`. See [internals/06](./internals/06-project-and-persistence.md).

### Device chain
The ordered set of devices on an `AudioUnit` — `MidiDeviceChain` (MIDI effects → instrument) and `AudioDeviceChain` (effects → sends → channel strip → output). Rewires during `ProcessPhase.Before`. See [internals/05](./internals/05-devices-and-effects.md).

## E

### Effect
A processor inserted in the audio chain that transforms its input — Compressor, Reverb, Delay, etc. Built from a box + adapter + processor triple. See [Ch. 11](./11-effects.md), [internals/05](./internals/05-devices-and-effects.md).

### Engine
The audio brain: the `EngineProcessor` running in the `AudioWorklet` plus the supporting workers. Exposed to the main thread through `EngineFacade`. See [internals/01](./internals/01-engine-processor.md).

### `EngineFacade`
The main-thread observable wrapper around the engine — `isPlaying`, `position`, `bpm`, `play()`, `stop()`, `setPosition()`. What your UI subscribes to. See [Quick Start](./quick-start.md), [internals/03](./internals/03-cross-thread-protocols.md).

### `engineToClient`
The typed RPC channel from the audio worklet back to the main thread — `log`, `error`, `fetchAudio`, `fetchSoundfont`, `notifyClipSequenceChanges`. See [internals/03](./internals/03-cross-thread-protocols.md).

## F

### Field
A typed slot on a box. `PrimitiveField<V>` (Float32, Int32, Boolean, String, ByteArray), `PointerField<P>`, `ArrayField<F>`, `ObjectField<F>`, or a placeholder `Field<P>` (no value, can be pointed at). See [Ch. 04](./04-box-system-and-reactivity.md).

### Fade-in / fade-out
A region-level gain envelope at the start or end of the clip, shaped by `Curve.normalizedAt(t, slope)`. Non-destructive. See [Ch. 09](./09-editing-fades-and-automation.md).

### Forge
The box-schema code generator (`@opendaw/lib-box-forge`). Reads schema files, emits TypeScript box classes into `packages/studio/boxes/src/`. Run after every schema change. See [internals/02](./internals/02-box-system.md), [internals/07](./internals/07-dev-workflow.md).

### Freeze
Bouncing an `AudioUnit` to an offline-rendered audio buffer that replaces the live chain in playback. Used when a chain is too DSP-heavy for real-time. See [internals/05](./internals/05-devices-and-effects.md#special-processors), [internals/06](./internals/06-project-and-persistence.md).

## H

### `HRClock`
A high-resolution timing channel between the audio thread and a dedicated Worker that wakes on `Atomics.notify`. Used for the DSP-load meter. See [internals/03](./internals/03-cross-thread-protocols.md).

## I

### `initializeOpenDAW`
The setup helper that wires `AnimationFrame`, the worker, the worklet, the `AudioContext`, and the sample manager — about 120 lines of plumbing that this repo copies into every demo. See [Quick Start](./quick-start.md).

### Instrument
A processor that *produces* audio in response to note events — `Tape`, `Vaporisateur`, `Soundfont`, `Apparat`, etc. Lives as the input device of an `AudioUnit`. See [Ch. 11](./11-effects.md), [internals/05](./internals/05-devices-and-effects.md).

## L

### Live broadcaster
The high-frequency streaming channel for peak meters, spectrum, waveform. Worklet → main, separate from `SyncStream`. See [internals/03](./internals/03-cross-thread-protocols.md).

### Loop offset / loop duration
On `AudioRegionBox`, the audio bounds within the source file that get tiled when the region's `duration` is longer than the file. See [Ch. 05](./05-samples-peaks-and-looping.md).

## M

### Main thread
The browser's UI thread — React renders, observable callbacks fire, `editing.modify()` opens box transactions. Cannot block; cannot run audio. See [Ch. 00](./00-system-architecture.md).

### `Messenger`
The typed `MessagePort` wrapper that all cross-thread RPC sits on top of. One per port, multiplexes channels via `channel(name)`. See [internals/03](./internals/03-cross-thread-protocols.md).

### Modular device
A device whose behaviour is defined by user-supplied code (Apparat, Spielwerk, Werkstatt) plus a graph of modules and connections stored in the box. See [internals/05](./internals/05-devices-and-effects.md).

### Musical timebase
A clip mode in which the clip's duration is measured in PPQN — stretches/shrinks with BPM. Contrast with Seconds timebase. See [Ch. 02](./02-timing-and-tempo.md).

## N

### `NoteEventCollectionBox`
The container for MIDI note events that a `NoteRegionBox` references. See [Ch. 08](./08-recording.md).

### `NoteRegionBox`
A region that holds MIDI note events. See [Ch. 04](./04-box-system-and-reactivity.md), [Ch. 08](./08-recording.md).

## O

### `.od`
The OpenDAW project file extension. Stored at `projects/v1/{uuid}/project.od` in OPFS. Inside, it's exactly an encoded `ProjectSkeleton` (magic header `"OPEN"` + version + box graph binary). See [internals/06](./internals/06-project-and-persistence.md).

### `Observable<T>`
An `lib-std` primitive with `.getValue()`, `.subscribe(observer)`, and `.catchupAndSubscribe(observer)`. The standard way SDK state is exposed. See [Ch. 03](./03-animation-frame.md).

### OPFS
Origin Private File System. The browser's per-origin sandboxed file system where OpenDAW persists projects, samples, and presets. See [internals/04](./internals/04-sample-loading.md), [Ch. 13](./13-troubleshooting.md).

### `Option<T>`
The `lib-std` discriminated-union wrapper around a possibly-absent value: `Some(value)` or `None`. Has `.map`, `.match`, `.unwrap`, `.ifSome`. Used for any field that may or may not be present.

## P

### Peaks
A multi-resolution min/max summary of an audio file used to render the waveform at any zoom level cheaply. Computed once at import, cached as `peaks.bin` next to the audio file. See [Ch. 05](./05-samples-peaks-and-looping.md), [internals/04](./internals/04-sample-loading.md).

### `PeaksPainter`
The canvas-rendering helper from `@opendaw/lib-fusion`. Takes a `Peaks.Stage` and draws it. Always call `peaks.nearest(unitsPerPixel)` first to pick the right zoom level. See [Ch. 05](./05-samples-peaks-and-looping.md).

### `PointerField`
A field that points at another vertex. Validated against the target's `pointerRules.accepts` at `refer()` time. See [internals/02](./internals/02-box-system.md).

### `PointerHub`
The "incoming" side of pointers — every box has one and you subscribe to it to know "what's pointing at me right now". See [internals/02](./internals/02-box-system.md).

### PPQN
Pulses Per Quarter Note. OpenDAW's unit of musical time. `PPQN.Quarter = 960`, so one quarter note is 960 pulses at *any* BPM. See [Ch. 02](./02-timing-and-tempo.md).

### Process phase
`ProcessPhase.Before` / `ProcessPhase.After` — the two notification points around the per-quantum render loop. Effect chains rewire in `Before`. See [internals/01](./internals/01-engine-processor.md).

### `Project`
The top-level owner of everything in a session: box graph, editing, sample manager, engine facade, mixer, automation, freeze. Three factories: `Project.new`, `Project.load`, `Project.loadAnyVersion`. See [internals/06](./internals/06-project-and-persistence.md).

### `ProjectSkeleton`
The minimal `{boxGraph, mandatoryBoxes}` payload that travels between disk, the main thread, and the audio worklet. Encoded with a `"OPEN"` magic header. See [internals/06](./internals/06-project-and-persistence.md).

## Q

### Quarter
`PPQN.Quarter = 960` — the constant that defines OpenDAW's musical-time resolution. Never changes; never depends on BPM. See [Ch. 02](./02-timing-and-tempo.md).

### `queryLoadingComplete()`
The `EngineCommands` RPC that returns a `Promise<boolean>` once every pending resource (sample, soundfont, WASM) has finished loading. `await` this before `play()` to avoid silent playback. See [Ch. 13](./13-troubleshooting.md).

## R

### `refer`
The `PointerField` method that sets the target. Validates against `pointerRules.accepts` and throws on type mismatch. The inverse is `defer()`. See [internals/02](./internals/02-box-system.md).

### Region
The box-graph word for a clip on a track. `AudioRegionBox`, `NoteRegionBox`, `ValueRegionBox`. See *Clip*.

### Render quantum
128 audio frames — the unit of work the audio thread does per `process()` call. At 48 kHz that's ~2.67 ms of audio, with ~2.9 ms of wall time to compute it. See [internals/01](./internals/01-engine-processor.md).

### Resource type
A flag on a box (`"preserved"` / `"internal"` / `"shared"`) controlling whether the UUID survives copy/paste and whether the box's children get followed during dependency walks. See [internals/02](./internals/02-box-system.md).

### `RingBuffer`
A single-producer / single-consumer ring over `SharedArrayBuffer` used by the recording processor to ship audio frames out of the worklet. See [internals/03](./internals/03-cross-thread-protocols.md).

## S

### Sample
An audio file plus its decoded `AudioData`, peaks, and metadata. Identified by a content-addressed UUID. See [Ch. 05](./05-samples-peaks-and-looping.md), [internals/04](./internals/04-sample-loading.md).

### `SampleLoader`
The main-thread handle on a sample's loading state. Observable; transitions through `progress → loaded | error`. See [Ch. 05](./05-samples-peaks-and-looping.md), [internals/04](./internals/04-sample-loading.md).

### Seconds timebase
A clip mode in which the clip's duration is measured in seconds — stays constant when BPM changes. Contrast with Musical timebase. See [Ch. 02](./02-timing-and-tempo.md).

### `SharedArrayBuffer`
A typed-array buffer shared between threads. Used by `SyncStream`, the HRClock, `RingBuffer`, and the engine state ring. Requires the page to be cross-origin isolated. See [Ch. 12](./12-browser-compatibility.md), [internals/03](./internals/03-cross-thread-protocols.md).

### Slope
The single parameter that shapes a `Curve` between linear (`0.5`), logarithmic (< `0.5`), and exponential (> `0.5`). See [Ch. 09](./09-editing-fades-and-automation.md).

### Stem
A single track or sub-mix rendered to its own output file during export, instead of bouncing everything to one stereo mix. Configured via `ExportConfiguration.stems`. See [Ch. 10](./10-export.md).

### `SyncLog`
The hash-chained transaction history kept alongside a project. Used for crash recovery. Not the primary save format — that's the `.od` snapshot. See [internals/06](./internals/06-project-and-persistence.md).

### `SyncStream`
The lock-free `SharedArrayBuffer`-backed state ring the worklet writes once per render quantum (position, BPM, meters, DSP load) for the main thread to poll. See [internals/03](./internals/03-cross-thread-protocols.md).

## T

### Take
One recorded pass over a loop region. Stored on a separate `TrackBox` within the same `AudioUnitBox`. Comp lanes let the user assemble a final take from several. See [Ch. 08](./08-recording.md).

### Tempo automation
Time-varying BPM, stored as `ValueEvent`s on the timeline's tempo track. See [Ch. 02](./02-timing-and-tempo.md).

### `TempoGridCursor`
A reusable cursor that holds integration state for incremental tempo-map calls. Available as a standalone export for custom analysis; most consumers benefit from its memoisation transparently via `tempoMap.intervalToSeconds()`. See [Ch. 02](./02-timing-and-tempo.md).

### Tempo map
`project.tempoMap` — the API for converting between PPQN and seconds in the presence of tempo automation. Without automation, equivalent to constant-BPM math. See [Ch. 02](./02-timing-and-tempo.md).

### Timebase
The unit a clip's `duration` is measured in — *Musical* (PPQN, BPM-dependent in seconds) or *Seconds* (constant in seconds, varies in PPQN). See [Ch. 02](./02-timing-and-tempo.md).

### `TimeInfo`
The tiny mutable struct on the audio thread holding the current position, transporting flag, recording flag, count-in flag, leap flag. Source of truth for "where is the playhead". See [internals/01](./internals/01-engine-processor.md).

### `TrackBox`
A lane inside an `AudioUnit` that holds regions. One `AudioUnit` can contain several `TrackBox`es (e.g. for take-based recording). See [Ch. 04](./04-box-system-and-reactivity.md).

### Transaction
A `BoxEditing.modify(callback)` call. Every mutation must happen inside one. Sequence: open → mutations → constraint check → commit → notify subscribers. Provides undo/redo. See [Ch. 04](./04-box-system-and-reactivity.md), [internals/02](./internals/02-box-system.md).

### Transient
An onset (drum hit, note attack) in audio. Detected per file via the three-band Linkwitz-Riley detector and stored as `TransientMarkerBox`es on the `AudioFileBox`. See [internals/04](./internals/04-sample-loading.md).

### Transport
The playback controller — play, stop, record, position, count-in. `project.engine` is the SDK surface; `TimeInfo` is the worklet state. See [internals/01](./internals/01-engine-processor.md).

## U

### UUID
A 16-byte identifier (typed `UUID.Bytes`). All boxes are keyed by UUID. **Use `UUID.newSet`/`UUID.newMap`** — native `Set`/`Map` compare by reference and won't deduplicate two byte-identical UUIDs. See [internals/02](./internals/02-box-system.md).

## V

### `ValueEventCollectionBox`
A box that holds a sorted collection of `ValueEvent`s — used for tempo automation, parameter automation, and per-region modulation. See [Ch. 09](./09-editing-fades-and-automation.md).

### `ValueMapping`
The numeric mapping attached to an automatable parameter (e.g. `ValueMapping.linear(-60, 0)` for a dB volume). Defines how a normalised `[0, 1]` automation value maps to natural units. See [internals/02](./internals/02-box-system.md).

### `VaryingTempoMap`
The concrete `TempoMap` implementation that accounts for tempo automation events. Exposed via `project.tempoMap`. See [Ch. 02](./02-timing-and-tempo.md).

## W

### Web Worker
A browser background thread without DOM access. OpenDAW uses several: peaks worker, OPFS worker, HRClock worker, FFmpeg worker, offline render worker. See [Ch. 00](./00-system-architecture.md), [internals/03](./internals/03-cross-thread-protocols.md).
